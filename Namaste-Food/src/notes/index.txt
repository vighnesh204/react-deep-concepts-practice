🚀 2 types routing in web apps 
- Client Side Routing
- Server Side Routing


🚀 Server Side Routing
=> server side routing means 
=> you have a index.html, about.html, contact.html
=> if i click on my anchor tag "/about.html" it reloads the whole page
=> it sends the network call to about.html and fetches that html and renders it on to the web page 
=> This is server side routing


🚀 Over here this is Client side routing
=> we are not making any network call 
=> while we are moving towards the page
=> because all the components are already loaded in our app
=> when i load the app for the first time it already has the code for about us 
=> when i go to about us page it just loads that component
=> it does not make any network call
=> the only network call is made when i make a network call in the API
=> I am not fetching a new page thats why this is known as Client side routing
=> This is the explanation for single page application also
=> Why It is a single page application ?
=> Its just one page just the components getting interchanged via Client side routing
=> we are implementing client side routing
